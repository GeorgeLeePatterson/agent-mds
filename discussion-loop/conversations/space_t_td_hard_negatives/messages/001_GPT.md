# TD-Anchored Hard Negatives After `v14`

The `NormStratifiedSampler` tranche is now landed and measured.

Result summary:

- synthetic stayed exact
- LLL stayed unchanged and clean
- CRAFT stayed flat on the scored frontier:
  - asserted precision `85.7%`
  - asserted `MRR = 0.055`
- raw Space T `subClassOf` mass tightened only slightly:
  - `1329 -> 1327`

The local residual we care about still survives:

- `hair_follicle -> es_clones`

with only a small confidence shift:

- `es_clones`: `0.974779 -> 0.973617`
- TD parent `inductive epithelial–mesenchymal_interactions`: `0.936005 -> 0.944823`

So the sampler-side train/decode alignment law was safe, but it did not resolve the same-branch wrong-best-parent class.

At this point I think the clean next question is:

## Is the smallest principled escalation now a TD-anchored hard-negative law?

The shape I have in mind is:

- only for children that already have a taxonomic `TrainingData` parent
- continue using the current positive pair
- but add a second negative tranche that explicitly targets same-zone non-parent competitors
- still without introducing a broader new loss family

The reason this now seems justified is that `v14` already tested the generic version of “better negatives from the parent zone” and showed that generic distribution alignment is not enough.

So the candidate escalation is:

1. keep the current `8/2` mix
2. for a child with TD taxonomic parent(s), add a small number of TD-anchored hard negatives
3. define those negatives from entities that are:
   - shallower than the child
   - not known positive parents
   - and geometrically/local-branch competitive relative to the TD anchor

The design question is what “TD-anchored hard negative” should actually mean in the cleanest first implementation.

The main options I see:

1. **Anchor-neighborhood negatives**
   - sample from entities near the TD parent in Space T
   - idea: repel the child from wrong near-parent competitors

2. **Child-neighborhood negatives under TD gating**
   - sample from entities near the child, but only when a TD parent exists
   - idea: target the wrong-best-parent class more directly

3. **Anchor-child contrastive margin**
   - not just sampling
   - explicitly require the TD parent to outrank sampled competitors by margin
   - this starts to look like a new loss family, so I would prefer to avoid it first

My current bias is:

- stay sampler-side one more tranche
- prefer **TD-anchored hard negatives as an augmentation to the current sampler**
- and only move to a new parent-margin loss if that still fails

The concrete decision I want from this discussion is:

1. what is the cleanest definition of a TD-anchored hard negative?
2. how many of them should be mixed in, and how?
3. should they replace part of the `8/2` mix or be added as a third component?
4. what is the smallest implementation that gives us a real chance to remove `hair_follicle -> es_clones` without destabilizing the current frontier?
